# TextCNN для классификации статей Habr.com

> Классификация статей с Хабра по уровню сложности с помощью свёрточной нейронной сети на текстах.

## Источник данных

| Параметр | Значение |
|---|---|
| **Сайт** | [habr.com](https://habr.com) |
| **Раздел** | AI и ML → [Машинное обучение](https://habr.com/ru/hubs/machine_learning/articles/) |
| **Инструмент сбора** | Playwright (headless Chromium) + BeautifulSoup |

## Разметка датасета

Разметка **автоматическая**: React-артефакт отправляет статьи из JSON-файла в Claude API батчами и сохраняет результат.

Классификация по **уровням сложности**, определяется по тегу на Habr:

| Метка | Уровень |
|:---:|---|
| `0` | 🟢 Простой |
| `1` | 🟡 Средний |
| `2` | 🔴 Продвинутый |

## Токенизация

Используется подход **word-level**.

Почему?

Тексты строгие, объёмные и выверенные (статьи). 
При character-level подходе свёртка «видела» бы буквы, а не термины — пришлось бы использовать гораздо большие ядра и терять интерпретируемость.

### Почему TextCNN?

| Преимущество | Описание |
|---|---|
| **Устойчивость к длине** | Слой `max_pool1d` схлопывает временну́ю размерность (`seq_len`), превращая текст любой длины в фиксированный вектор — будь то короткая заметка или лонгрид |
| **Скорость** | В отличие от RNN (LSTM/GRU), CNN обрабатывает слова **параллельно**, что делает обучение в разы быстрее — критично при обучении на CPU или слабых GPU |
| **Выделение триггеров** | Модель отлично вылавливает маркеры сложности: фразы вроде `"математическое ожидание"`, `"ассемблерный код"` или `"memory leak"` задают «хардкорность» статьи быстрее, чем рекуррентная сеть, анализирующая контекст последовательно |

### Пайплайн

```
[Текст статьи]
      │
      ▼

      │
      ▼
 Permute          [B, 300, L]
      │
      ▼

      │

      │
      ▼

      │
      ▼

      │
      ▼
[Выходные логиты классов]
```

## Гиперпараметры

| Параметр | Значение | Описание | Обоснование |
|:---|:---|:---|:---|
| `vocab_size` | зависит от датасета | Размер словаря токенизатора | Рекомендуется top-50 000 токенов; редкие слова → `[UNK]` |
| `embedding_dim` | `300` | Размерность вектора эмбеддинга | Стандарт для предобученных векторов (*FastText*, *Word2Vec* от RusVectores) |
| `filter_sizes` | `[3, 4, 5]` | Размеры ядер свёртки (`kernel_size`) | Извлекают n-граммы разной длины: три-, четырёх- и пятиграммы |
| `n_filters` | `128` | Количество фильтров на каждый размер ядра | Баланс между точностью и скоростью |
| `dropout` | `0.5` | Коэффициент регуляризации Dropout | Высокое значение критично против переобучения на длинных и зашумлённых текстах |
| `output_dim` | `3` | Количество выходных классов | Простой / Средний / Продвинутый |
| `pad_idx` | `int` | Индекс токена заполнения (`[PAD]`) | Выравнивание длины батчей; градиенты через `[PAD]` не считаются |

## Разделение данных

Исходный датасет (`markup_dataset.xlsx`) проходит предобработку, разбивается на выборки в пропорции **70 / 15 / 15** со стратификацией по метке сложности.

```python
train_df, rest_df = train_test_split(df, test_size=0.30, random_state=42, stratify=df['difficulty'])
val_df, test_df  = train_test_split(rest_df, test_size=0.50, random_state=42, stratify=rest_df['difficulty'])
```

| Выборка | Назначение |
|---|---|
| `train.csv` | Обучение модели |
| `val.csv` | Подбор гиперпараметров и ранняя остановка |
| `test.csv` | Финальная оценка качества |

### Предобработка перед разбивкой

Перед разбивкой из датасета удаляются строки с пропусками и тексты, в которых доля кириллицы ниже 50% (галлюцинации модели-разметчика — польские слова вместо русских).

## Результаты

### Базовая итерация (7 эпох, dropout = 0.5)

| Метрика | Значение |
|---|---|
| **Accuracy (test)** | 44% |
| **Val Accuracy** | ~42% |

**Classification Report (тест):**

| Класс | Precision | Recall | F1-score |
|:---:|:---:|:---:|:---:|
| 0 — Простой | — | 0.10 | — |
| 1 — Средний | 0.26 | 0.25 | ~0.26 |
| 2 — Продвинутый | наивысший | наивысший | наивысший |

Матрица ошибок показывает явное доминирование класса 2: модель склонна предсказывать его даже для примеров классов 0 и 1.

### Улучшенная итерация (15 эпох, dropout = 0.4, взвешенный CrossEntropy)

| Метрика | Базовая | Улучшенная |
|---|---|---|
| **Accuracy (test)** | 44% | 47% |
| **Recall класса 0** | 0.10 | 0.38 |
| **F1 класса 2** | — | 0.66 |
| **Recall класса 1** | 0.25 | 0.00 ⚠️ |

> Взвешивание классов улучшило распознавание классов 0 и 2, однако привело к игнорированию класса 1.

---

## Выводы

**Что сработало:**
- Быстрый рост точности на train-выборке (>90% к концу обучения).
- Взвешивание классов улучшило recall для классов 0 и 2.

**Проблемы:**
- Сильное переобучение: train accuracy ≈ 100%, val/test accuracy ≈ 42–47%.
- Несбалансированность классов приводит к тому, что модель предсказывает преимущественно класс 2.
- После добавления весов класс 1 (Средний) перестаёт распознаваться вовсе.

**Следующие шаги:**

1. **Предобученные эмбеддинги** — инициализировать слой `Embedding` векторами FastText/Word2Vec с RusVectores вместо случайных весов.
2. **Балансировка данных** — применить SMOTE или андерсэмплинг мажорного класса.
3. **Тонкая настройка весов** — менее агрессивное взвешивание классов, чтобы не «забыть» класс 1.
4. **Лемматизация** — привести слова к базовой форме для уменьшения словаря и улучшения обобщения.
5. **Ранняя остановка** — прекращать обучение при отсутствии улучшения val loss, чтобы снизить переобучение.
